# Design Context — OpenAkita Visual Language

> Extracted from the OpenAkita codebase as a portable design standard.
> Apply this system to any project that needs a clean, warm, technically-sophisticated feel.

---

## Users

Tech-savvy individuals who enjoy exploring AI tools. They value:
- **Clarity over decoration** — they want to see what's happening, not be sold to
- **Keyboard and workflow efficiency** — they use tools daily, not once
- **Transparency** — they want to understand what the system is doing
- **Tasteful defaults** — they appreciate good design but don't want to configure it

Context of use: personal productivity, creative work, technical exploration. Often running the tool alongside editors, terminals, and browsers — the UI must feel at home in a dense workspace.

---

## Brand Personality

**Three words:** Precise. Warm. Forward.

| Dimension | Expression |
|-----------|-----------|
| Precise | Every pixel has intent. No decorative noise. Information density without clutter. |
| Warm | Soft edges, comfortable spacing, approachable type. Never cold or clinical. |
| Forward | Subtle glass effects, refined motion, modern color usage. Feels like the near future, not sci-fi. |

**What this is NOT:**
- NOT enterprise OA — no dense toolbars, no status bar overload, no corporate blue
- NOT ChatGPT — not a single-column chat bubble interface; structured work surfaces with spatial layout
- NOT flashy SaaS — no gradient salads, no 3D illustrations, no hero sections with floating devices

---

## Design Principles

1. **Quiet confidence** — The interface doesn't shout. It works. Whitespace is a feature, not waste. Let content breathe.

2. **Glass, not chrome** — Use translucency and depth (backdrop-filter, soft shadows) to create hierarchy. Never use heavy borders, raised bevels, or strong gradients to separate layers.

3. **Round but not bubbly** — Generous border-radius (12-16px on surfaces) creates warmth without looking toy-like. Buttons at 14px feel clickable, not childish.

4. **Motion as meaning** — 150ms transitions for state changes. Animations communicate (loading, success, error), never decorate. If reduced-motion is active, everything still works.

5. **Dark mode as first-class citizen** — Design dark theme alongside light, not as an afterthought. Use the same token system, same contrast ratios, same emotional tone.

## Visual Language

### Color System

#### Gray Scale (Light → Dark)

Use a **Slate-tinted gray** (not pure gray, not blue-gray, not warm-gray). Tailwind Slate is the canonical source.

| Role | Light | Dark |
|------|-------|------|
| Page background | `#f4f5f7` | `#0f172a` |
| Card/surface | `#ffffff` | `#1e293b` |
| Elevated surface | `rgba(255,255,255,0.88)` with blur | `rgba(30,41,59,0.88)` with blur |
| Border | `rgba(17,24,39,0.12)` | `rgba(255,255,255,0.10)` |
| Text primary | `rgba(17,24,39,0.95)` | `rgba(241,245,249,0.95)` |
| Text secondary | `rgba(17,24,39,0.55)` | `rgba(148,163,184,0.80)` |
| Text tertiary | `rgba(17,24,39,0.40)` | `rgba(100,116,139,0.70)` |

#### Brand Color

Primary: **`#2563eb`** (blue-600). Use as accent, interactive states, and brand moments.

| Variant | Value | Usage |
|---------|-------|-------|
| Brand | `#2563eb` | Buttons, links, active states |
| Brand hover | `#1d4ed8` | Hover states |
| Brand light | `#60a5fa` | Dark mode brand |
| Brand glow | `rgba(37,99,235,0.15)` | Focus rings, subtle highlights |
| Brand gradient | `linear-gradient(135deg, #2563eb, #38bdf8)` | Hero/icon accents only |

Per-product brand colors can replace the blue while keeping the entire rest of the system.

#### Semantic Colors

### Typography

#### Font Stacks

```css
--font-body: "Noto Sans SC", ui-sans-serif, system-ui, -apple-system,
  "Segoe UI", "Microsoft YaHei", "PingFang SC", sans-serif;

--font-brand: "Noto Serif SC", ui-serif, Georgia, serif;

--font-mono: "Cascadia Code", "Fira Code", "JetBrains Mono", "Consolas", monospace;
```

For non-CJK projects, drop the CJK-specific families. Keep the system-ui fallback chain.

#### Type Scale

| Role | Size | Weight | Letter-spacing | Line-height |
|------|------|--------|---------------|-------------|
| Page title | 28px | 700 | 0.5px | 1.2 |
| Section title | 22px | 700 | — | 1.2 |
| Card title | 15px | 700 | 0.1px | 1.3 |
| Body | 14px | 400 | 0.2px | 1.5 |
| Label | 12px | 600 | — | 1.4 |
| Caption/help | 11px | 400-500 | — | 1.4 |
| Code | 13px | 400 | — | 1.6 |

Always set `-webkit-font-smoothing: antialiased`.

### Spacing

4px base grid. Common values:

| Value | Usage |
|-------|-------|
| 4px | Tight gaps (icon-to-text, inline spacing) |
| 8px | Standard small gap (form fields, list items) |
| 12px | Default gap (card margins, section spacing) |
| 14px | Content padding |
| 16px | Large padding (panels, sections) |
| 24px | Modal/dialog padding |
| 32px | Page-level spacing |

---

### Border Radius

| Element | Radius |
|---------|--------|
| Cards, panels | 12px |
| Buttons (primary) | 14px |
| Buttons (secondary) | 10px |
| Inputs | 8px |
| Modal/dialog | 16px |
| Navigation items | 10px |
| Pills, badges, avatars | 999px |
| Code blocks | 8px |

Key principle: **surfaces > 12px, interactive elements 8-14px, decorative pills 999px**.

---

### Shadows

| Level | Value | Usage |
|-------|-------|-------|
| Subtle | `0 1px 2px rgba(0,0,0,0.04)` | Cards resting state |
| Surface | `0 8px 32px rgba(17,24,39,0.08)` | Panels, sidebar |
| Brand glow | `0 4px 24px rgba(37,99,235,0.15)` | Focused/active elements |
| Elevated | `0 24px 64px rgba(0,0,0,0.18)` | Modals, popovers |

Dark mode: increase opacity by ~2-3x on the same shadow definitions.

---

### Glass Effect

```css
.glass-panel {
  background: rgba(255, 255, 255, 0.88);  /* dark: rgba(30,41,59,0.88) */
  backdrop-filter: blur(16px);
  border: 1px solid rgba(17, 24, 39, 0.12);  /* dark: rgba(255,255,255,0.10) */
  border-radius: 12px;
}
```

Use for sidebar, main panels, floating elements. NOT for inline content or form fields.

---

### Background Texture

Optional dot-matrix pattern for page backgrounds:

```css
body {
  background-color: var(--bg-app);
  background-image: radial-gradient(var(--matrix-dot) 1px, transparent 1px);
  background-size: 24px 24px;
  /* --matrix-dot: rgba(17,24,39,0.06) light / rgba(255,255,255,0.06) dark */
}
```

Adds subtle depth without visual noise. Remove if the project calls for flat backgrounds.

### Motion

| Property | Duration | Easing | Usage |
|----------|----------|--------|-------|
| Color/opacity | 150ms | ease | Hover states, text color changes |
| Transform | 150ms | ease | Icon rotations, scale on hover |
| Layout | 200ms | ease | Sidebar collapse, grid resize |
| Modal enter | 200ms | ease-out | slideUp animation |
| Loading pulse | 1200ms | ease-in-out infinite | Processing indicators |

**Rules:**
- Never exceed 200ms for interactive feedback
- Loading animations should be 1.2-1.5s cycles
- Always implement `prefers-reduced-motion: reduce` — kill all animation and blur

### Icons

**Library:** Lucide (stroke style, `stroke-width: 1.5-2`, `viewBox: 0 0 24 24`)

| Size | Usage |
|------|-------|
| 16px (size-4) | Inline with text, badges |
| 18px (size-[18px]) | Navigation items |
| 20px (size-5) | Button icons, card headers |
| 24px (size-6) | Standalone, empty states |

No emoji in UI. No filled icon variants. Icons inherit `currentColor`.

### Component Patterns

#### Button
```
Primary:   bg: var(--brand), color: #fff, radius: 14px, padding: 10px 12px
Secondary: transparent bg, 1px border, radius: 10px, padding: 8px 24px
Small:     radius: 6px, padding: 5px 12px, font-size: 12px
Icon:      28×28px, radius: 6px
```

#### Input
```
Height: 36px, radius: 8px, border: 1px solid var(--border), padding: 4px 12px
Focus: border-color: var(--brand), box-shadow: 0 0 0 1px var(--bg), 0 0 8px var(--brand)
```

#### Card
```
bg: var(--card-bg), border: 1px solid var(--border), radius: 12px
padding: 12px, shadow: 0 1px 2px rgba(0,0,0,0.04)
```

#### Modal
```
Overlay: rgba(15,23,42,0.5), backdrop-filter: blur(6px)
Content: width 480px, radius 16px, padding 24px
Shadow: 0 24px 64px rgba(0,0,0,0.18)
Animation: slideUp 200ms ease-out
```

#### Status Pill
```
radius: 999px, padding: 2px 8px, font-size: 11px, font-weight: 600
Colors: semantic background (e.g. #dcfce7) + semantic text (e.g. #166534)
```

#### Navigation Item
```
radius: 10px, min-height: 38px, padding: 8px 10px
Active: bg var(--nav-active), left border accent, color var(--brand)
```

### Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Color contrast | WCAG AA minimum (4.5:1 for text, 3:1 for UI) |
| Focus visibility | 2-3px brand-colored ring on all interactive elements |
| Reduced motion | Kill all animation and blur via media query |
| Color blindness | Daltonized theme variants (deuteranopia, protanopia) |
| Touch targets | Minimum 36px height on all interactive elements |
| Font scaling | Rem-based sizing where possible, min 11px |

---

### Dark Mode Strategy

1. **Same token system** — swap values via `[data-theme="dark"]` selector
2. **Slate 800/900** as dark base — never pure `#000000` or `#111111`
3. **Lower opacity** for borders and dividers in dark (`0.10` vs `0.12`)
4. **Increase shadow opacity** 2-3x
5. **Brand color shifts lighter** — `#2563eb` → `#60a5fa`
6. **Semantic colors shift lighter** — maintain same perceptual brightness
7. **Glass panels** invert to dark translucent surfaces

---

### How to Apply to a New Project

1. **Copy the token system** — CSS variables for colors, spacing, radii, shadows
2. **Replace brand color** — swap `#2563eb` with your product's primary
3. **Keep the gray scale** — Slate-tinted grays work for any modern tool
4. **Keep the component patterns** — buttons, cards, inputs, modals are framework-agnostic
5. **Adopt the motion rules** — 150ms transitions, reduced-motion respect
6. **Implement glass selectively** — sidebar + main panels, not everywhere
7. **Add the dot texture** if the project benefits from subtle depth
8. **Configure dark mode tokens** from day one, not as an afterthought
